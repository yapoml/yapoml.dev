Introduction

Yapoml generates page objects for your UI tests on the fly. Describe pages and components with a concise YAML syntax — Yapoml produces fluent, type-safe C# code as you save the file. The result: less boilerplate, more readable tests, and a tighter feedback loop for both Selenium and Playwright.

At a glance

  • ✍️ Describe pages in a few lines of YAML — no class scaffolding, no glue code.
  • ⚡ Get strongly-typed C# page objects automatically, ready to use via .Ya().
  • 🧪 Express expectations and interactions fluently, with full IntelliSense support.

Quick Start​

3 easy steps to get started right now. Let's create a real test which verifies that on the https://www.nuget.org/packages?q=yapoml page there is at least 1 package, and that all packages have a description and tags.

❶ Install NuGet package​

  • Selenium
  • Playwright

Install Yapoml.Selenium NuGet package.

Install Yapoml.Playwright NuGet package.

❷ Define page​

Add new Packages.page.yaml file which defines the layout of the page. Read more about how to define pages and components.

Packages.page.yaml
url:
  path: https://www.nuget.org/packages
  query:
    - q

packages: 
  by: .package

  description: .package-details
  tags: .package-tags a

❸ Interact with it​

The page is automatically transformed to C# types available for interactions immediately. Explore what interactions and expectations you can use on the page.

Program.cs
driver.Ya().PackagesPage.Open(q: "yapoml")
  .Packages.Expect(it => it.IsNotEmpty().Each(package =>
    {
      package.Description.IsNotEmpty();
      package.Tags.IsNotEmpty();
    })
  );
